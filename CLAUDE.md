## Build & Run

**Regenerate the Xcode project** (required after editing `project.yml`):
```bash
xcodegen generate
```

**Build for simulator** (verify no compile errors):
```bash
xcodebuild -project Trigger.xcodeproj -scheme Trigger \
  -destination 'generic/platform=iOS Simulator' -configuration Debug build
```

**Install and run on a specific simulator**:
```bash
# Find available simulators
xcrun simctl list devices available | grep iPhone

# Boot, install, and launch
xcrun simctl boot <DEVICE_ID>
xcrun simctl install <DEVICE_ID> path/to/DerivedData/.../Trigger.app
xcrun simctl launch <DEVICE_ID> com.trigger.reminder

# Take a screenshot
xcrun simctl io <DEVICE_ID> screenshot out.png
```

**Launch on a specific tab** (debug convenience, uses `CommandLine.arguments`):
```bash
xcrun simctl launch <DEVICE_ID> com.trigger.reminder --tab-reminders   # tab index 1
xcrun simctl launch <DEVICE_ID> com.trigger.reminder --tab-stats        # tab index 2
```

There are no automated tests in this project. After any code change, run the `xcodebuild` command above and confirm `** BUILD SUCCEEDED **`.

## Architecture

The app uses **SwiftUI + `@Observable`** (iOS 17+) with a single central store injected via `.environment(store)` at the root.

### Data flow
```
TriggerApp (@main)
  └─ AppStore (@Observable, @MainActor)   ← single source of truth
       ├─ [Reminder]                       ← persisted to UserDefaults as JSON
       ├─ UserStats (XP, streak, achievements)
       └─ NotificationService.shared       ← called inside Task {} blocks
```

Views access the store with `@Environment(AppStore.self)`. All mutations go through `AppStore` methods (`addReminder`, `completeReminder`, `deleteReminder`, `snoozeReminder`). Never mutate `AppStore` state directly from a view.

### Notification pipeline
`AppDelegate` (UIApplicationDelegate) is the `UNUserNotificationCenterDelegate`. When a user taps a notification action ("DONE", "SNOOZE_1H", "SNOOZE_3H"), `AppDelegate` posts an `NSNotification` (`.reminderCompleted` / `.reminderSnoozed`). `TriggerApp` listens for these via `.onReceive` and forwards them to `AppStore`.

Each `Reminder` stores its own `notificationIdentifier` (a UUID string) used to cancel/reschedule via `NotificationService`.

### Persistence
`AppStore` saves both `[Reminder]` and `UserStats` as JSON-encoded `Data` in `UserDefaults` under the keys `trigger_reminders_v1` and `trigger_user_stats_v1`. The `_v1` suffix allows future schema migrations without conflicting with old data.

On load, new achievements from `Achievement.all` are merged into the stored stats so that new achievement types added in code are automatically available to existing users.

### XP & Gamification
- Create reminder → +10 XP
- Complete on time (within 1h of trigger) → +25 XP; late → +15 XP
- Achievement unlock → +50 XP bonus
- Levels 1–10 defined by thresholds in `UserStats.level` computed property
- Streak increments once per calendar day in `updateStreakForToday()`, called at app launch

### Project configuration
The Xcode project is **generated by xcodegen** from `project.yml`. Do not edit `Trigger.xcodeproj/project.pbxproj` by hand — edit `project.yml` and re-run `xcodegen generate`. Swift language mode is set to `"5.0"` to avoid Swift 6 strict concurrency enforcement; all view models are `@MainActor` for thread safety.

### Design system
All colors live in `Utils/AppColors.swift` as `Color` extensions (prefix `trigger*`). Use only these named colors and `LinearGradient.triggerOrangeGradient` / `.triggerCardGradient` — never hardcode raw color values in views. The app is locked to `.dark` color scheme.
